# Restaurant Sales Analytics Dashboard

## Project Overview
This project analyzes restaurant sales data to uncover trends in orders, revenue, menu performance, and customer behavior.  
The analysis was conducted using **SQL** for data exploration and **Power BI** for interactive visualization, with insights presented in a professional PowerPoint report.

---

## Objective
The goal of this project is to:
- Evaluate restaurant performance over time
- Identify top selling items and categories
- Analyze pricing and customer ordering behavior
- Provide data-driven recommendations to improve revenue and operations

---

## Tools & Technologies
- **SQL (MySQL)** – Data querying and analysis
- **Power BI** – Data modeling, KPIs, dashboards
- **PowerPoint** – Executive presentation
- **GitHub** – Project version control and portfolio hosting

---

## Dataset
The analysis is based on two main tables:

- **menu_items**: menu item id, item name, category, price  
- **order_details**: order details ID, order ID, date, time, item ID

## Key Insights
- Sales show consistent peaks at the beginning and middle of each month.
- Asian cuisine leads in item volume, while Italian dishes generate higher revenue.
- Mid-priced items perform best in terms of demand.
- High-value orders tend to include premium-category items.

---

## Recommendations
- Run promotions during end of month low demand periods.
- Focus marketing efforts on top selling categories and items.
- Reevaluate pricing for underperforming high priced items and make coupons.
- Optimize inventory and staffing based on demand patterns.
